What do these results mean?

The LRS Test evaluates whether an LRS meets the technical requirements of the xAPI specification.

A successful LRS Test means that:

  • the LRS successfully completed the tests run by a specific version of the LRS Test Suite
    there is reasonable certainty that the LRS correctly implements the LRS requirements in the xAPI specification
  • there is reasonable certainty that the LRS will behave as expected when handling xAPI requests from learning record producers (LRP) and learning record consumers (LRC)

This does not mean that:

  • the LRS is endorsed by the ADL Initiative, DoD, or Federal Government
  • the LRS is guaranteed to have been tested for defects in all functionality
  • the LRS has been evaluated against or is otherwise certified to meet cybersecurity requirements
